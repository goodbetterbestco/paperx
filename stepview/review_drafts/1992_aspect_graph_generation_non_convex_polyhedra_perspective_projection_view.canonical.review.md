## 1 INTRODUCTION

The effective object representation of the three-dimensional (3D) objects is an essential problem for the major research area of 3D object recognition. Three-dimensional object recognition is basically a matching problem between the two-dimensional (2D) image information and the prestored model. The prestored model can be either a 3D object model or multiple 2D views of the objects. Here, we introduce an algorithm to generate effective multiple 2D views of the 3D objects so that the recognition process is reduced to a much simpler 2D to 2D matching problem. The idea behind this approach is that if a stored view is similar to the image, the recognition process will be simple and fast. The set of 2D views, which represent the infinite number of possible views, need to be as small as possible to minimize the number of views that must be stored. Therefore, the views are generally taken to be topologically distinct. This paper is concerned with the creation of aspect graphs for use as object models in a 3D object recognition system. Koenderink and van Doorn \({ }^{(1)}\) introduced the notion of an aspect graph for representing object shape. An aspect is a qualitatively distinct view of an object as seen from connected viewpoints in the viewpoint space. We will develop a method for directly creating the aspect graph of a polyhedral

Each arc in the graph indicates an adjacency between two aspects and the plane, which separates the two associated viewing cells.

The aspect graph representation combines an exact 3D model of an object with a set of generic views. Each generic view is associated with a viewing cell of space from which it can be seen. Thus, the aspect graph is a hybrid representation, which combines the completeness of a viewpoint-independent representation with the simplicity of a viewpoint-dependent representation. The aspect graph of an object is a graph structure in which (1) each node of the graph, or aspect, represents a topologically distinct view of the object as seen from some maximal, connected region of viewpoint space, and there is a node for each aspect; (2) each arc represents a visual event, which occurs at the transition from one region of the viewpoint space to a neighboring region, and there is an arc for each such transition. There are two approaches to partition the viewing space: (1) uniform object-independent partition and (2) aspect graph approach. The viewing space has been modeled as either Gaussian sphere, resulting in an orthographic projection aspect, or as all 3D space, resulting in a perspective projection aspect. In the first approach the viewing space is partitioned uniformly by projecting a tessellated regular polyhedron onto the sphere. The tessellation of the polyhedron and the partition of the regions on the surface of the sphere is predetermined and independent of the structures. Then, the neighboring regions that have the same aspect are combined. \({ }^{(2-4)}\) The uniform partition has to be fine enough to avoid missing any of the aspects. The advantage of this approach is that the partition is easy to compute and fix for all objects. The difficulty is in selecting the right scale of the partition. Under a fine partition, the views in many neighboring regions are qualitatively similar. On the other hand, under coarse partition some important views might be missing. In the second approach, the viewing space, as proposed by Koenderink and van Doorn, \({ }^{(1)}\) is partitioned according to the qualitative structure of the view, which is called the aspect. An aspect may be defined in one of several ways, depending on the application at hand. The aspect may be defined by the graph structure of the line drawing, by the set of visible faces, by the set of occluding edges, or by the other structure of the view. They describe a graph structure called the "visual potential" of the object. Chakravarty and Freeman describe an approach to object recognition, which uses similar concepts called "characteristic views". \({ }^{(5)}\) Some previous work has been published on constructing the aspect graph. Based on different viewing methods (orthographic or perspective projections) as well as different viewing objects (convex or non-convex objects), we may classify the previous aspect graph generation methods into four categories:

(1) Perspective-convex. Stewman and Bowyer \({ }^{(6)}\) first proposed a solution for generating the aspect graphs of the planar-face objects from the perspective view. Gaultieri et al. \({ }^{(7)}\) showed how to construct an aspect graph for convex polyhedra with the viewpoint constrained to lie in the plane containing the polygon, using perspective projections. Watts \({ }^{(8)}\) described an \(O\left(n^{4}\right)\)-time algorithm for enumerating the "principal views" of convex polyhedra.

- Perspective-non-convex. Stewman and Bowyer, (9) in their later paper, described an algorithm for constructing the viewing data of convex polyhedra under perspective projection. Wang and Freeman 10) presented a method to partition the viewing space of a planar-faced solid into characteristic view domains and found the set of characteristic views of the solid.

- Orthographic-convex. Mckenna and Seidel'!!) constructed minimal and maximal shadows of convex polyhedra in the plane using the \(O(nº)\) algorithm. Edelsbrunner et al. (12) developed similar methods.

When dealing with the convex polyhedra, the parcellation of viewing space is entirely determined by the planes on which the face of the object lies. The major complication in handling the non-convex polyhedra is the possibility of self-occlusion. Stewman and Bowyer(9) introduced the auxiliary surfaces for parcellation of the viewing space of the non-convex polyhedra. An auxiliary plane is defined by the visual interaction of a vertex and an edge of the object. Wang and Freeman 10) followed the same algorithm and renamed the three parcellation surfaces as type A, type B, and type C surfaces. However, the parcellation surfaces (types B and C) are not sufficient for describing all EE and EEE event transitions. Here, we analyze all the possible occurrences of EE and EEE events and develop algorithms for constructing their corresponding viewing cells. Our approach is different from previous methods in developing a robust and comprehensive aspect graph generation algorithm.

## 2 FACIAL EVENT AND EE EVENT

One of the major steps in aspect graph generation is to find the occurrences of singularities. The singularities are determined by three basic events: facial events, \(E E\) events, and \(E E E\) events. The facial event occurs when the viewer changes his viewpoints and the entire face becomes invisible. Also, when a T-junction appears when edge A occludes edge B or not, the \(E E\) event occurs. When the \(E E E\) event occurs, there will be three edges which occlude one another. A detailed description about the three events will be described later. In this section, we develop several concepts for finding the viewing cell in 3D viewpoint space from which the specific event (facial or \(E E\) event) can be observed. The differences between our approaches and the others are that we do a complete categorization of different types of \(E E\) events and use them to handle more complex non-convex objects. In the next section, we study \(E E E\) events and add these events to our system. Before mentioning the facial and \(E E\) events, we define some phrases that will be used throughout this paper.

Visual event. A visual event is a viewpoint where the connectivity, or the topology, of the projected line drawing changes. The visual event arises when the viewpoint moves from one aspect to another aspect.

Viewing cell. It is a set of continuous viewpoints in the viewing space from which the same aspect is observed. The range of the viewing cell for an event depends on the definition of the aspect.

Expanding plane. The plane in which one face lies on is called the expanding plane for that face.

Basic plane (BP). The expanding plane of the object face is called the basic plane of the object.

Auxiliary plane (AP). The auxiliary plane is a plane generated by a vertex and an edge of an object.

## 4 ALGORITHMS

Given all the parcellation planes above, we can partition the viewing space into the event cell for that particular event. Every aspect is composed of a different combination of events. For a certain aspect, the corresponding aspect cell is intersection of the respective event cells. If the intersection of the respective event cell exist, the aspect can be perceived from the viewpoints in that aspect cell. The aspect is composed of different combinations of features: (1) a number of the visible faces; (2) the total number of visible faces; (3) a number of \(E E\) events for the partially occluded faces; (4) the positions of the \(E E\) events on the respective face; (5) the number of \(E E E\) events for two occluded faces; (6) the positions of the \(E E E\) events on the respective two faces. To test all the possible intersections of all the event cells is very complicated and computation expensive. The alternative way to find the possible intersections is through the test points. The test points are found to cover all the possible existing aspect cells. From the test point validation process, we may find all the existing aspect cells.
